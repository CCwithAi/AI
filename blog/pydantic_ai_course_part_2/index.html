Mastering Pydantic Logfire: A New Framework for Observability

I'm excited to share my experience with Logfire, a new type of observability platform built on Open Telemetry by the same team behind Pantic AI. With its amazing support for Python, Logfire is a game-changer for building AI agents. In this post, I'll walk you through how to set up and use Logfire, highlighting its key features and benefits.

Why Logfire Matters

When building AI agents, reliability is crucial. Unfortunately, Large Language Models (LLMs) are notoriously unreliable, making them the "worst databases" as the Pantic team put it. This is where Logfire comes in – providing a solid tracing and debugging system that can save hours of troubleshooting time. By setting up dashboards and alerts, you can monitor and react to emerging issues before they cause disruptions or outages.

Getting Started with Logfire

To set up Logfire, I created a developer account, installed the SDK with a simple command, authenticated with Logfire, and started sending data. The process is quite easy, and the documentation provides great examples to get you started.

Configuring Logfire

After setting up my project in Logfire, I explored the user interface, which includes dashboards, alerts, and settings. I created a new project, followed the setup instructions for development, and installed Logfire with a pip install command. Next, I authenticated with Logfire, created a Tomel file, and specified my project.

Hello World Example

To test my setup, I created a simple Python script that sent an info statement to Logfire. The script consisted of just three lines of code, importing Logfire, configuring it, and sending the data. I was thrilled to see the "hello world" message successfully logged on the Logfire website.

Using Spans and Log Levels

In the next examples, I explored Logfire's span feature, which allows me to organize my logs by context. I created a span, logged a message, and set an attribute. On the Logfire website, I could see my span and the contained log message. I also experimented with different log levels, including notice, info, debug, warning, error, and fatal. Logfire's color-coding system makes it easy to distinguish between these levels.

Logging Exceptions

To demonstrate how to log exceptions, I created a try-catch block and raised a ValueError. On the Logfire website, I could see the exception marked in red, making it easy to identify and prioritize. I also used SQL filters to isolate exceptions and fetch specific log messages.

Instrumenting Methods

Finally, I learned how to instrument certain methods using Logfire's instrumentation feature. By adding a directive to my method, I could log entire method calls without having to manually create spans. This feature is incredibly powerful for debugging and tracing issues.

Conclusion

In this post, I've provided a brief overview of Logfire's key features and demonstrated how to set it up and use it with Python. Logfire's simplicity, Python-centric design, and great Pantic integration make it an attractive choice for building AI agents. With its support for structured data and SQL queries, Logfire is a valuable tool for any developer working with AI. I hope this post has inspired you to give Logfire a try and explore its many features.

SECTION STEPS

**Getting Started with Logfire**

  1. Create a developer account at Logfire.
  2. Install the Logfire SDK using a simple command.
  3. Authenticate with Logfire and set up a project.
  4. Start sending data to Logfire.

**Setting Up Logfire**

  1. Navigate to the Logfire website and sign up with your GitHub account or use other authentication methods.
  2. Create a new project by giving it a name.
  3. Explore the project setup, including links for your account and project.
  4. Set up your alerts and go to the settings.
  5. Change the description, but not the project name.
  6. Delete the project and follow instructions for development and production.
  7. Navigate to the project and explore the user interface.

**Hello World Example**

  1. Install Logfire using `pip install logfire`.
  2. Authenticate with Logfire using `logfire off`.
  3. Create a Tomel file under your user.
  4. Use `logfire project use TS` to set up the project.
  5. Import Logfire and call `logfire.Doc.configure`.
  6. Ascent the data with `logfire.doLog` and send an info statement.

**Span Feature Example**

  1. Create a span using `logfire.span`.
  2. Organize logs by tool or context within the span.
  3. Set the attribute `result` with the result data.
  4. Log the result data using `logfire.doLog`.
  5. Navigate to the Logfire dashboard to see the span and info log.

**Log Levels Example**

  1. Log different levels (notice, info, debug, warning, error, and fatal).
  2. Observe the different levels on the Logfire dashboard.
  3. Use different coloring to distinguish between levels.

**Logging Exceptions Example**

  1. Log exceptions using a try-catch block.
  2. Catch the exception and log it using Logfire.
  3. Explore the exception on the Logfire dashboard.
  4. Use SQL to filter exceptions and fatal errors.

**Instrumenting Methods Example**

  1. Use `logfire.instrument` to instrument a method.
  2. Define the method and pass a message.
  3. Run the method and observe the logs on the Logfire dashboard.
  4. Explore the row data to see the method call and response.

Summary and Next Steps

Logfire is a new framework from the same team as Pantic AI. It's simple to use, Python-centric, and has great Pantic integration. It uses open Telemetry and supports structured data. It allows SQL queries for filtering and debugging. Next, we'll explore parsing text and getting structured outputs from Pantic AI.
